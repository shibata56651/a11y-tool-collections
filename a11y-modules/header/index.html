ヘッダー実装時の注意点

ヘッダーの基本的なセマンティック構造

<header>：サイトまたはセクションの導入部分を定義

<h1>：サイト名やサービス名を含む最上位見出し

<nav>：メインナビゲーション領域の明示

ランドマーク：スクリーンリーダーでのページ構造理解を支援

ドロップダウンメニューの実装

階層的な情報を効率的に表示するドロップダウンメニューのアクセシブル実装：

重要なARIA属性：

• aria-expanded：サブメニューの開閉状態（true/false）

• aria-haspopup：サブメニューの存在を示す（true）

• aria-controls：操作対象のサブメニューのID

• role="menu"：サブメニュー領域のロール

• role="menuitem"：各メニュー項目のロール

ハンバーガーメニューの実装

モバイルデバイスでのメニュー表示を制御するハンバーガーメニューボタン：

重要な実装ポイント：

• aria-label：ボタンの目的を明確に説明

• aria-expanded：メニューの開閉状態を動的に更新

• aria-controls：制御するメニューのIDを指定

キーボードナビゲーション

ヘッダーナビゲーションでのキーボード操作実装：

対応すべきキー操作

Tab / Shift+Tab：メニュー項目間のフォーカス移動

Enter / Space：リンクの実行、ドロップダウンの開閉

Escape：開いたドロップダウンを閉じる

矢印キー：ドロップダウン内でのフォーカス移動

Home / End：メニューの最初・最後への移動

フォーカス管理

フォーカストラップ：モバイルメニュー表示時のフォーカス制御

フォーカス復帰：ドロップダウンを閉じた際の元位置への復帰

視覚的表示：フォーカス状態の明確な表示

テスト・検証方法

ヘッダー実装後の確認項目：

スクリーンリーダー：ナビゲーション構造とラベルが正しく読み上げられるか

キーボード操作：すべての機能がキーボードのみで操作可能か

フォーカス表示：現在位置が視覚的に明確に表示されるか

レスポンシブ動作：異なるデバイスサイズでの適切な表示・操作

パフォーマンス：読み込み速度と動作の滑らかさ

参考：実装例の確認

このページ上部のヘッダー実装が実際の例です。以下の操作で機能を確認してみてください：

• キーボードのTabキーでメニュー項目を移動

• ドロップダウンメニューをキーボード（Enter、Escape）で操作

• スクリーンリーダーでナビゲーション構造を確認

• モバイルサイズでハンバーガーメニューの動作確認
